# איפיון ראשוני — אפליקציית גלובוס טיסות (שם זמני: "Mor Airlines")

> **הערה ל‑Claude שיבנה:** המסמך הזה מגדיר את ה‑*מה*, ה‑*למה*, ואת האילוצים הקשיחים. יש לך חופש מלא על ה‑*איך* (ארכיטקטורת קומפוננטות, ניהול state, חלוקת קבצים) כל עוד אתה נשאר בתוך האילוצים והעקרונות למטה. במקומות שמסומנים "מומלץ" — זו הצעה, לא דרישה.

---

## 1. מטרה ותמצית

אפליקציית WEB אישית, מתנה עבור מור, שהופכת את ה"בוא ניסע למשהו אחר" החוזר שלה לטקס קטן ומרגש. בכל פעם שהיא מדברת על לטוס, פותחים את האפליקציה, לוחצים כפתור, וגלובוס תלת‑ממד מסתחרר, נועל יעד, וצולל אליו עם פין שנופל — ואז נשלף כרטיס עלייה למטוס מעוצב עם פרטי היעד.

מעבר לרגע ההגרלה, האפליקציה היא **דרכון חי משותף שלכם**: מקומות שמפנטזים עליהם, מקומות שדקרנו כי *ממש* רוצים, מקומות שכבר סגרנו, ומקומות שכבר היינו בהם (עם הערה אישית וזיכרון). הגלובוס מספר את כל הסיפור — איפה היינו, לאן אנחנו טסים, ועל מה אנחנו חולמים. הדרכון מסונכרן בין שני הטלפונים, כך שלשניכם אותו דרכון אחד (ראה 7.x — שכבת הסנכרון).

המטרה הרגשית: פנטזיה + זיכרון + "מה הלאה", בעטיפה חגיגית. המטרה התפעולית: כלי שלעולם לא "נגמר", כי בכל שלב אפשר להוסיף ולערוך.

## 2. עקרונות מנחים (לא לסטות מהם)

- **טקס ההגרלה קדוש.** הרגע מהלחיצה ועד כרטיס העלייה צריך להיות חד, חלק, ובלי המתנות. בלי spinners, בלי קריאות רשת על המסלול הקריטי. כל מה שצריך כדי להגריל ולחשוף יעד נקרא מ‑localStorage (מקומי, מיידי). הסנכרון ל‑Supabase רץ תמיד ברקע — לעולם לא חוסם את הטקס.
- **עברית קודם, RTL.** הממשק כולו בעברית, `dir="rtl"`, פונטים עבריים. אנגלית רק היכן שמתאים (קודי שדה, שם היעד באנגלית על הכרטיס).
- **חגיגי וטקטילי.** סאונד, קונפטי, פין שנופל, כרטיס ש"מודפס" ומחליק פנימה. הפרטים הקטנים הם המוצר.
- **אישי.** נוסעים מור + אייל, מושב 2A ליד החלון, הערות אישיות. זה שלהם, לא גנרי.
- **תמיד ניתן להרחבה.** הוספה ועריכה של יעדים זמינות בכל שלב, מתוך הממשק, בלי לגעת בקוד.

## 3. תרחישי שימוש

1. **הרגע הספונטני** — היא אומרת "בוא ניסע", שולפים טלפון, לוחצים, הגלובוס מגריל יעד, כרטיס עלייה קופץ. ~30 שניות.
2. **דפדוף ברשימה שלה** — בלי הגרלה: עוברים על היעדים, צוללים לכל אחד, קוראים (מנה מקומית, עונה, "למה דווקא פה").
3. **סגרנו באמת** — מסמנים יעד כ‑BOOKED, הוא מקבל חותמת בדרכון וקשת מלאה על הגלובוס.
4. **כבר היינו** — מסמנים יעד כ‑VISITED, מוסיפים תאריך והערה אישית/זיכרון.
5. **דקירה** — מקום שמפנטזים עליו במיוחד מקבל פין שננעץ בגלובוס (בולט מנקודות החלום הרגילות).
6. **יעד חדש** — היא זורקת מקום שלא ברשימה, לוחצים "+", מקלידים שם, והוא נכנס (קואורדינטות מתקבלות אוטומטית).
7. **סנכרון בין שניכם** — היא דוקרת מקום בנייד שלה; כשאתה פותח אצלך, זה כבר שם.

## 4. פיצ'רים מרכזיים

### 4.1 הגלובוס (בית)
גלובוס תלת‑ממד שמסתובב לאט בחלל. ישראל מסומנת כ"בית". כל יעד מוצג לפי המצב שלו (ראה 4.4) בצבע/סמן מובחן. לחיצה על יעד פותחת אותו; כפתור מרכזי "לאן טסים?" מפעיל את ההגרלה.

### 4.2 טקס ההגרלה (הלב של האפליקציה)
רצף קבוע:
1. **Idle** — גלובוס מסתובב לאט, נקודות החלום מנצנצות.
2. **לחיצה** → צליל "בינג‑בונג" של הכרזת שדה, התאורה מסביב מתעמעמת.
3. **סחרור** → ה‑auto‑rotate מאיץ כמו גלגל מזל, תיפוף עולה.
4. **נעילה** → המצלמה צוללת אל הקואורדינטות, הגובה יורד.
5. **נפילת פין** → פין נופל עם קפיצה קטנה + "דינג".
6. **חשיפה** → שם היעד נכנס ענק, קונפטי, קשת טיסה נמתחת מישראל ליעד.
7. **כרטיס עלייה** → מחליק מלמטה (ראה 4.3).
8. **פעולות** → "טוס שוב" / "שמור לדרכון" / "ספר לי עוד" / שיתוף.

ההגרלה היא **בחירה רנדומלית מתוך הרשימה**, לא יצירה. אפשר לסנן לפי וייב לפני ההגרלה ("בא לי חוף"). מומלץ לשקלל כך שיעדי חלום שעוד לא ביקרנו בהם יקפצו יותר.

### 4.3 כרטיס העלייה למטוס
מציג: נוסעים מור + אייל, מושב 2A (חלון), שם היעד (עברית + אנגלית), המנה המקומית, העונה הכי טובה, שורת "למה דווקא פה", ותמונה + תקציר קצר מ‑Wikipedia. פעולות: טוס שוב, שמור לדרכון, ספר לי עוד (פותח את התקציר המלא מ‑Wikipedia), שיתוף.

### 4.4 הדרכון והמצבים
לכל יעד יש מצב, ויזואלי על הגלובוס + חותמת בדרכון:
- **חלום (dream)** — נקודה מנצנצת.
- **דקור (starred)** — פין שננעץ בגלובוס, בולט (heart/pin). זהו דגל מעל מצב החלום, לא מצב נפרד — יעד יכול להיות "חלום + דקור".
- **סגור (booked)** — סמן מלא + קשת מישראל + חותמת "BOOKED", עם תאריך טיסה אם יש.
- **היינו (visited)** — סמן בצבע "עבר" + קשת בצבע שונה + חותמת עם תאריך + **הערה אישית/זיכרון**.

הדרכון הוא מסך נפרד שמרכז את כל היעדים לפי מצב, עם החותמות.

### 4.5 הוספה ועריכה (CRUD) — זמין בכל שלב
- **הוספה**: מקלידים שם מקום → הקואורדינטות מתקבלות אוטומטית (geocoding), תמונה + תקציר מ‑Wikipedia. שאר השדות אופציונליים למילוי.
- **עריכה**: כל שדה בכל יעד ניתן לעריכה — כולל יעדי הזרע (seed). שינוי מצב, הוספת/עריכת הערה אישית, דקירה/ביטול דקירה.
- **מחיקה** של יעדים שהוספו.

כל שינוי נשמר מיד ל‑localStorage ונדחף ברקע ל‑Supabase (ראה 7.x).

### 4.6 פילטרים
לפי וייב (חוף / עיר / הרפתקה / אוכל / רחוק / קרוב) ולפי מצב/סקופ.

## 5. מודל הנתונים

סכמת יעד מוצעת (TypeScript):

```ts
type DestinationState = 'dream' | 'booked' | 'visited';
type VibeTag = 'beach' | 'city' | 'adventure' | 'food' | 'far' | 'near';

interface Destination {
  id: string;
  nameHe: string;
  nameEn: string;
  lat: number;
  lng: number;
  tagline?: string;        // עברית
  localDish?: string;      // עברית — המנה המקומית
  bestSeason?: string;     // עברית — העונה הכי טובה
  whyHere?: string;        // עברית — שורה אחת
  vibeTags: VibeTag[];
  state: DestinationState;
  starred: boolean;        // "דקור" — מפנטזים במיוחד
  travelDate?: string;     // עבור booked
  visitedDate?: string;    // עבור visited
  personalNote?: string;   // זיכרון/הערה (בעיקר ל‑visited)
  imageUrl?: string;       // cache מ‑Wikipedia
  wikiSummary?: string;    // cache מ‑Wikipedia
  source: 'seed' | 'user';
  updatedAt: string;
}
```

**ארכיטקטורת השכבות (חשוב):** יש שלוש שכבות שצריך למזג:
1. **Seed** — `destinations.ts` מקובץ בקוד, ~40–60 יעדים מוקפדים. בסיס קבוע, לקריאה.
2. **User (localStorage)** — כל מה שהמשתמש עושה: יעדים שהוסיף, שינויי מצב, דקירות, הערות, ועריכות ל‑seed. זוהי שכבת ה‑**source of truth לתצוגה** — ממנה הטקס וכל המסכים נקראים מיידית.
3. **Sync (Supabase)** — שכבת ה‑User כולה מסונכרנת כ‑**בלוב JSON משותף יחיד** ל‑Supabase, כך ששני בני הזוג חולקים דרכון אחד (פירוט מלא ב‑7.x).

עריכות ל‑seed נשמרות כ‑**overrides** בשכבת ה‑User שמרובדים מעל קובץ הזרע בזמן טעינה (seed + overrides = המצב המוצג). כך הקובץ המקורי נשאר נקי וניתן לעדכן אותו בעתיד בלי לדרוס את העריכות של המשתמש. מומלץ לאחד את כל שכבת ה‑User במקום אחד (ראה stack), וזה גם מה שמסתנכרן.

## 6. מקורות נתונים

- **`destinations.ts` (seed)** — תוכן מוקפד שייכתב מראש (לא נוצר בזמן ריצה). קואורדינטות מוזנות ידנית ומדויקות.
- **Wikipedia / Wikimedia REST API** — תמונה + תקציר לפי שם מקום. חינם, בלי מפתח. נשמר ב‑cache בשכבת ה‑User כך שלא קוראים פעמיים.
- **Open‑Meteo Geocoding API** (מומלץ) — הופך שם של מקום שהוקלד לקואורדינטות. חינם, בלי מפתח. משמש רק במסך ההוספה.
- **Supabase** — שכבת הסנכרון של הדרכון המשותף (ראה 7.x).
- **localStorage** — ה‑source of truth לתצוגה ולטקס; cache מקומי שמסתנכרן ל‑Supabase.

**מה במפורש אין:** אין SQLite, אין LLM בזמן ריצה, אין login/חשבונות, ואין מפתחות API פרטיים בקוד. הקריאות ל‑Wikipedia ול‑geocoding הן keyless. Supabase משתמש ב‑anon key הציבורי (בטוח לחשיפה בצד לקוח), מוגן ב‑RLS + מזהה דרכון לא‑נחיש. כל קריאת הרשת מחוץ למסלול הקריטי של ההגרלה.

## 7. סטאק טכנולוגי

- **React + TypeScript + Vite** — אפליקציית client.
- **react-globe.gl** — הגלובוס (עטיפה מעל three.js / three-globe). הסיבוב, צלילת המצלמה (`pointOfView`), הקשתות והנקודות — דרך ה‑API שלה.
- **framer-motion** — אנימציית ה‑UI/chrome: כרטיס העלייה, מעברי מסכים, ופינים כ‑HTML שמוצמדים לגלובוס (`htmlElementsData`). **לא** מנפיש את הגלובוס עצמו — זה three.js.
- **canvas-confetti** — קונפטי.
- **Howler.js** או HTML5 Audio (מומלץ Howler) — צלילים: בינג‑בונג, תיפוף, דינג, מחיאות כפיים.
- **Tailwind CSS** — עיצוב, עם תמיכת RTL.
- **vite-plugin-pwa** — PWA מתקין עם אייקון על מסך הבית.
- **ניהול state**: מומלץ `zustand` עם `persist` middleware ל‑localStorage (נקי לאיחוד seed + overrides + נתוני משתמש). לבחירתך.
- **Supabase** — שכבת הסנכרון (ראה 7.x).
- **דיפלוי**: Vercel.

### 7.x שכבת הסנכרון (Supabase) — local-first

**עיקרון על:** local-first. localStorage הוא ה‑source of truth לתצוגה; Supabase הוא שכבת סנכרון ברקע. קר‑סטארט של ה‑DB לעולם לא חוסם את הטקס — נושך לכל היותר בטעינה הראשונה אי‑פעם על מכשיר חדש (כשאין עדיין cache).

**מודל הנתונים ב‑Supabase — בלוב יחיד, לא סכמה.** מכיוון שזה שני אנשים שחולקים dataset קטן אחד, אין צורך בטבלאות/auth/realtime. טבלה אחת:

```
passports (
  id          uuid primary key,   -- מזהה הדרכון המשותף (לא נחיש)
  data        jsonb,              -- כל שכבת ה‑User מסריאלית כאן
  updated_at  timestamptz
)
```

שורה אחת לדרכון המשותף. כל ה‑state (יעדים שהוספו, overrides, מצבים, דקירות, הערות) יושב ב‑`data`.

**"Auth" של עניים.** אין login. הגישה דרך ה‑anon key הציבורי + RLS מופעל, והאבטחה נשענת על `id` (UUID) לא‑נחיש שמשותף רק בין שניכם (נשמר ב‑env / localStorage / לינק חד‑פעמי). מספיק לחלוטין לאפליקציית גאג לשניים. אם אי‑פעם תרצה הגנה אמיתית — Supabase Auth, מחוץ לסקופ.

**זרימת הסנכרון:**
- **בטעינה:** קוראים מ‑localStorage → מציגים מיד. ברקע: מושכים את הבלוב מ‑Supabase, ממזגים (last‑write‑wins לפי `updated_at` — מי שעדכני יותר מנצח), מעדכנים localStorage + UI.
- **בעריכה:** כותבים ל‑localStorage מיד (UI חד) → דוחפים את הבלוב המלא ל‑Supabase ב‑debounce עם `updated_at` חדש.
- **קונפליקטים:** last‑write‑wins על כל הבלוב מספיק ל‑v1 (נדיר ששניכם עורכים באותה שנייה). מיזוג ברמת השדה — אופציונלי, לא נדרש.

**Keep‑alive נגד pause.** ב‑free tier פרויקט נכנס ל‑pause אחרי שבוע חוסר פעילות, וה‑wake לוקח ~30 שניות. בגלל local-first זה כמעט לא מורגש, אבל מומלץ לחסל גם את זה עם GitHub Action מתוזמן (חינמי) שמריץ ping/select פשוט ~פעם בשבוע.

## 8. שלבי בנייה מוצעים

מומלץ לחלק ל‑`.md` נפרד לכל phase. אפשר לאחד/לסדר מחדש לפי שיקולך. הסנכרון נבנה אחרון בכוונה — האפליקציה עובדת מלא local-first בלעדיו, והוא שכבה תוספתית.

- **Phase 0 — שלד**: Vite + TS + Tailwind + RTL + דיפלוי ריק ל‑Vercel.
- **Phase 1 — גלובוס סטטי**: `destinations.ts` עם 10–15 יעדים, נקודות חלום, סמן בית (ישראל).
- **Phase 2 — טקס ההגרלה (הלב)**: סחרור → צלילה → פין → קונפטי + קשת → כרטיס עלייה. סאונד.
- **Phase 3 — דרכון, מצבים ו‑persistence מקומי**: dream / starred / booked / visited, חותמות, הערות אישיות, persistence ל‑localStorage (יסוד ה‑local-first).
- **Phase 4 — CRUD**: הוספה/עריכה/מחיקה, Wikipedia (תמונה+תקציר), Open-Meteo (קואורדינטות ליעד חדש), שכבת overrides.
- **Phase 5 — סנכרון Supabase**: טבלת `passports`, anon key + RLS + מזהה משותף, סנכרון בלוב ברקע (pull/merge בטעינה, debounced push בעריכה), keep‑alive.
- **Phase 6 — פילטרים, ליטוש, PWA**: פילטרי וייב/סקופ, סאונד סופי, אייקון, התקנה.

## 9. מחוץ לסקופ ל‑v1

- **"ספר לי עוד" מבוסס LLM חי** — v1 מסתפק בתקציר Wikipedia. גרסת LLM (Vercel serverless שמ‑proxy‑ת ל‑Anthropic) היא הרחבה עתידית, מחוץ למסלול הקריטי.
- **מצב "הפתע אותי מכל העולם"** — pool רנדומלי גדול מעבר לרשימה. v2.
- **חשבונות / login / multi‑user אמיתי.** הדרכון המשותף מאובטח ב‑UUID לא‑נחיש בלבד (בחירה מכוונת מינימלית). auth אמיתי דרך Supabase Auth — בהמשך אם יידרש.
- **Realtime / מיזוג ברמת השדה.** v1 משתמש ב‑pull‑on‑load + debounced push + last‑write‑wins.

## 10. החלטות פתוחות (עבורך)

1. **שם / מותג** — להישאר עם "Mor Airlines" כעטיפה, או כיוון אחר?
